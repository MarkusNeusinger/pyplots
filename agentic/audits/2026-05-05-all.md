## Summary
The repo's hands-on CI pipeline is broken (frontend lint fails with 32 errors, TypeScript type-check fails with 7 errors caused by a missing `react-force-graph-2d` package install) and a critical prompt-injection RCE path exists in `spec-create.yml` + `report-validate.yml` because issue templates auto-label and both workflows interpolate `${{ github.event.issue.body }}` directly into the Claude prompt without an `author_association` gate or a tool-list restriction. Production GCP posture has the highest concentration of high-severity findings (Cloud Run runs as the project-wide-editor compute SA, three production secrets never rotated, residential IPs whitelisted on Cloud SQL, zero alerting policies); the other big themes are catalog drift (15 specs created 2026-04-11 still have zero implementations and are missing from sitemap.xml; ~65% of spec YAMLs have `updated: null`) and observability gaps in the workflow-driven LLM pipeline (zero token/latency/cost logging around `prompt_claude_code` despite that path running every spec→impl→review attempt).
