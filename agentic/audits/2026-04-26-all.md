## Summary
The repo is in good *baseline* hygiene (ruff clean, format clean, strong test surface, well-designed workflow concurrency, solid Pydantic Settings module, robust SSRF defense in `api/routers/proxy.py`), but the audit surfaces three overlapping critical security findings concentrated in the Claude-driven GitHub Actions surface: any external user can induce Claude to act with `contents/issues/pull-requests: write` either by commenting `@claude` (`util-claude.yml` — no author gating) or by opening a templated issue (`spec-create.yml` / `report-validate.yml` interpolate `${{ github.event.issue.title|body }}` directly into Claude prompts), and a single `python3 -c` call in `impl-review.yml` inlines an attacker-controlled `title` field into a Python heredoc, achieving runner RCE. The high-severity bucket is dominated by LLM-pipeline gaps (no prompt caching, no SDK timeouts, hardcoded retired model strings, zero token/latency observability) plus three concrete frontend regressions (dead `HomePage.tsx`, 743-line `SpecTabs.tsx` with a module-level mutable cache, hardcoded RGBA overlays that break the new tri-state dark mode), one bona-fide MCP runtime bug (greenlet error reading deferred `Impl.code`), and DB↔model drift that `alembic check` already detects but no CI gate catches.
